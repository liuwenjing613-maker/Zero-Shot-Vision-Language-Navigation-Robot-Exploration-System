## 二、架构对比

### 2.1 v1 架构（基础版）

```
┌─────────────────────────────────────────────────┐
│                  v1 架构图                       │
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌─────────┐      ┌─────────────┐              │
│  │ System2 │ ───► │  云端 VLM   │              │
│  │ (异步)  │ ◄─── │  目标识别   │              │
│  └────┬────┘      └─────────────┘              │
│       │                                         │
│       ▼ 每 2s 更新一次                          │
│  ┌─────────────────────────────────────┐       │
│  │            System1 主循环            │       │
│  │  1. 获取 RGB+Depth                  │       │
│  │  2. 若有目标 → A* 规划              │       │
│  │  3. 直接跳到下一 waypoint           │       │
│  │  4. 可视化                          │       │
│  └─────────────────────────────────────┘       │
│                                                 │
└─────────────────────────────────────────────────┘
```

**特点**：
- System2 每 2s 向云端发一次请求
- 目标更新后直接 A* 规划，跳到下一个 waypoint
- **无防死锁、无探索、单目标**

```
│  │                 主循环                           │   │
│  │                                                  │   │
│  │  ┌──────────────┐    ┌──────────────────┐       │   │
│  │  │ FBE 前沿探索  │    │  短期记忆防死锁   │       │   │
│  │  │ Searching    │    │  5s 无进展 →     │       │   │
│  │  │ 过久 → 探索  │    │  黑名单 + 重规划  │       │   │
│  │  └──────────────┘    └──────────────────┘       │   │
│  │                                                  │   │
│  │  ┌──────────────────────────────────────┐       │   │
│  │  │ 成功判定: Target Locked + 深度近     │       │   │
│  │  │         + verify 验证通过           │       │   │
│  │  └──────────────────────────────────────┘       │   │
│  │                                                  │   │
│  │  ┌──────────────────────────────────────┐       │   │
│  │  │ 多目标: First X, then Y, finally Z  │       │   │
│  │  │ 依次完成，自动切换下一个目标         │       │   │
│  │  └──────────────────────────────────────┘       │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**新增**：
1. **FBE 前沿探索**：Searching 连续 350 帧（约 5s）无目标 → 前往 2-3m 外可达点探索
2. **短期记忆**：5s 内距离未缩短 0.15m → 目标加入黑名单，重新规划
3. **成功前验证**：深度近时请求云端 verify，避免红桌子当红衣服
4. **多目标序列**：支持 `"First find X, then Y, finally Z"` 格式

### 2.3 v3 架构（出生地环视 + 目标确认 + FBE 智能抑制）

```
┌─────────────────────────────────────────────────────────────┐
│                        v3 架构图                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌───────────────────────────────────────────────────────┐ │
│  │               出生地 4 视野环视 (新增)                  │ │
│  │                                                        │ │
│  │   视野0 ─► scan ─► conf=0.3                           │ │
│  │     │                                                  │ │
│  │   turn 90° (9步)                                       │ │
│  │     │                                                  │ │
│  │   视野1 ─► scan ─► conf=0.8 ← 最高!                   │ │
│  │     │                                                  │ │
│  │   turn 90° (9步)                                       │ │
│  │     │                                                  │ │
│  │   视野2 ─► scan ─► conf=0.2                           │ │
│  │     │                                                  │ │
│  │   turn 90° (9步)                                       │ │
│  │     │                                                  │ │
│  │   视野3 ─► scan ─► conf=0.1                           │ │
│  │     │                                                  │ │
│  │   转向视野1 (最高 conf) ─► 开始导航                    │ │
│  └───────────────────────────────────────────────────────┘ │
│                          │                                  │
│                          ▼                                  │
│  ┌───────────────────────────────────────────────────────┐ │
│  │                    主循环                              │ │
│  │                                                        │ │
│  │  ┌──────────────────────────────────────────────────┐ │ │
│  │  │ 目标完全确认 (target_fully_verified)              │ │ │
│  │  │                                                   │ │ │
│  │  │ 规划后请求 verify:                                │ │ │
│  │  │   - 通过 → target_fully_verified = True          │ │ │
│  │  │   - 不触发 FBE                                   │ │ │
│  │  │   - 撞墙时不随便转向                              │ │ │
│  │  │   - status=Searching 也不清空目标                 │ │ │
│  │  └──────────────────────────────────────────────────┘ │ │
│  │                                                        │ │
│  │  + FBE (仅未确认时触发)                               │ │
│  │  + 短期记忆                                           │ │
│  │  + 成功前验证                                         │ │
│  │  + 多目标序列                                         │ │
│  └───────────────────────────────────────────────────────┘ │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**v3 独有**：
1. **出生地 4×90° 环视**：出生后先原地转一周（4 个视野各 90°），对每个视野请求 scan 模式获取目标可能性，选择最高的方向再出发
2. **目标完全确认 (target_fully_verified)**：规划后立即 verify，通过则锁定目标，后续不再因 Searching 而清空、不触发 FBE、撞墙时仅微调
3. **FBE 智能抑制**：已完全确认目标后，即使暂时看不到目标也不触发 FBE，避免"明明在走却随便转向"

## 七、典型场景对比

### 场景 1：目标在背后

| 版本 | 行为 |
|------|------|
| v1 | 持续 Searching，无明确脱困策略 |
| v2 | 350 帧后触发 FBE，去探索点环视，可能找到目标 |
| v3 | **出生时就环视 360°**，大概率直接知道目标在哪个方向 |

### 场景 2：目标在拐角处，中途视野丢失

| 版本 | 行为 |
|------|------|
| v1 | 丢失后重新 Searching，可能偏离原路径 |
| v2 | 丢失后 Searching，可能触发 FBE 去探索 |
| v3 | **已完全确认目标 → 不清空路径、不触发 FBE**，继续沿原路径走 |

### 场景 3：目标误识别（红桌子当红衣服）

| 版本 | 行为 |
|------|------|
| v1 | 直接判定成功（误报） |
| v2 | 深度近时请求 verify，验证不通过则不计成功 |
| v3 | 同 v2 + 规划时就 verify，更早发现误识别 |

## 十、版本迁移指南

### v1 → v2
1. 替换 `local_robot_system1.py` 为 `local_robot_simple_v2.py`
2. 云端部署 `cloud_brain_server_v2.py`（支持 verify）
3. 配置多目标指令格式

### v2 → v3
1. 替换为 `local_robot_simple_v3.py`
2. 云端无需修改（scan 已支持）
3. 可调整 `DEGREES_PER_TURN` 确保转满一周
