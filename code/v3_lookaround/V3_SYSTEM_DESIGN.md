# VLN 导航系统 V3 - 系统设计文档

## 一、系统概述

V3 是一个基于视觉语言模型（VLM）的零样本视觉语言导航系统，采用**本地-云端分离架构**：
- **本地端**：运行 Habitat 仿真器，负责感知、决策和执行
- **云端**：运行 VLM（Qwen2.5-VL-7B），负责目标识别和推理

## 二、核心创新点

| 版本 | 创新点 |
|------|--------|
| V1 | 基础 VLM 导航 + 深度引导 |
| V2 | + FBE（前沿探索）+ 短期记忆防死锁 |
| **V3** | + 出生地环视 + 目标锁定验证 + 位置卡住检测 |

### 4.2 各阶段详解

#### 阶段1: 出生地环视（Spawn Scan）

**目的**：在起点进行 360° 环视，选择最可能存在目标的方向出发，避免一开始就走错方向。

```
步骤:
1. 捕获当前视角图像
2. 请求云端 scan 模式，获取目标置信度 (0~1)
3. 转向 90°（9步 × 10°/步）
4. 重复步骤 1-3，共 4 次
5. 选择置信度最高的方向
6. 转向该方向，进入主导航循环
```

**配置参数**：
- `SPAWN_SCAN_VIEWS = 4`（4个方向）
- `DEGREES_PER_TURN = 10`（每步转 10°）
- `TURNS_PER_90 = 9`（90° 需要 9 步）

#### 阶段2: 主导航循环

**云端状态**：
| 状态 | 含义 | 返回内容 |
|------|------|----------|
| `Target Locked` | 明确看到目标 | 像素坐标 (u, v) |
| `Inferred` | 推测目标方向 | 像素坐标 (u, v) |
| `Searching` | 未检测到目标 | 无坐标 |

**导航逻辑**：

```python
if status == "Target Locked":
    # 1. 将像素坐标转换为 3D 世界坐标
    # 2. 规划 A* 路径
    # 3. 请求 verify 验证目标
    # 4. 沿路径前进

elif status == "Inferred":
    # 1. 根据像素偏移计算转向
    # 2. 平滑转向（避免左右摇摆）
    # 3. 前方无障碍时前进

elif status == "Searching":
    # 1. 原地缓慢转向搜索
    # 2. 累计搜索帧数
    # 3. 超过阈值触发 FBE
```

## 五、关键机制详解

### 5.1 FBE（Frontier-Based Exploration）前沿探索

**触发条件**（任一满足即触发）：
1. **撞墙**：前方深度 < 0.6m 连续 4 帧
2. **搜索过久**：`Searching` 状态连续 350 帧（约 7 秒）
3. **位置卡住**：位置移动 < 0.3m 连续 300 帧（约 6 秒）

**执行流程**：
```
1. 在当前位置 2.5m 半径内随机采样可导航点
2. 筛选距离 > 1.2m 的点作为探索目标
3. 规划 A* 路径前往探索点
4. 到达后进行平滑环视（12 帧）
5. 退出 FBE，继续正常导航
```

**关键代码**：
```python
def get_explore_waypoint(sim, curr_pos):
    for _ in range(10):
        pt = sim.pathfinder.get_random_navigable_point_near(
            curr_pos, FBE_EXPLORE_RADIUS
        )
        if dist(pt, curr_pos) >= FBE_MIN_DISTANCE:
            return pt
    return None
```

### 5.2 目标锁定验证（Target Verification）

**目的**：防止将相似物体误认为目标（如红桌子当红衣服）。

**两阶段验证**：

```
阶段1: 规划时验证 (verify on plan)
┌─────────────────────────────────────────┐
│ 条件: status == "Target Locked"         │
│ 动作: 规划路径后立即请求 verify          │
│ 结果: target_fully_verified = True/False│
│ 效果: 若 True，后续不会因短暂 Searching  │
│       而触发 FBE，保持对目标的追踪       │
└─────────────────────────────────────────┘

阶段2: 成功前验证 (verify on success)
┌─────────────────────────────────────────┐
│ 条件: 深度 < 0.5m (已靠近)              │
│ 动作: 再次请求 verify 确认              │
│ 结果: 通过 → 目标达成                   │
│       未通过 → 清除目标，继续搜索        │
└─────────────────────────────────────────┘
```

### 5.3 短期记忆防死锁

**问题**：机器人可能在某个"疑似目标"附近徘徊，无法脱困。

**解决方案**：
```
每 5 秒检查一次:
  if 距离目标未缩短 0.15m:
      将该目标加入黑名单
      清除当前路径
      重新搜索
```

**黑名单机制**：
- 黑名单容差：0.5m 内的点视为同一目标
- 不会规划前往黑名单内的点
- 目标切换时清空黑名单

### 5.4 位置卡住检测

**问题**：即使云端返回 `Inferred`，机器人可能因物理障碍无法移动。

**检测逻辑**：
```python
每 100 帧检查:
    dist_moved = 当前位置 - 上次位置
    if dist_moved < 0.3m:
        stuck_frames += 100
    else:
        stuck_frames = 0
    
    if stuck_frames >= 300:  # 约 6 秒
        触发 FBE
```

## 六、像素到 3D 坐标转换

```python
def get_3d_point(u, v, depth_img, agent_state, sim):
    # 1. 获取深度值
    z_depth = depth_img[v, u]
    
    # 2. 相机内参反投影
    f = (IMG_WIDTH / 2) / tan(HFOV / 2)
    x_c = (u - cx) * z_depth / f
    y_c = -(v - cy) * z_depth / f
    z_c = -z_depth
    
    # 3. 相机坐标 → 世界坐标
    camera_pose = agent_state.sensor_states["color_sensor"]
    world_point = camera_pose.transform(x_c, y_c, z_c)
    
    # 4. Snap 到 NavMesh
    return sim.pathfinder.snap_point(world_point)
```

## 七、异步云端通信

```
┌─────────────────────────────────────────────────────────┐
│                    主线程 (导航循环)                     │
│  ┌─────────┐                                            │
│  │ 每帧更新 │ ──▶ shared_state.current_rgb              │
│  │ RGB+Depth│ ──▶ shared_state.current_depth            │
│  └─────────┘ ──▶ shared_state.new_image_ready = True    │
│       │                                                  │
│       ▼                                                  │
│  读取 shared_state.latest_goal_uv                        │
│  读取 shared_state.latest_status                         │
└─────────────────────────────────────────────────────────┘
                         ▲
                         │ 共享状态
                         ▼
┌─────────────────────────────────────────────────────────┐
│                Cloud Worker 线程                         │
│  while True:                                             │
│      if 距上次发送 > 0.6s and new_image_ready:          │
│          POST 图像到云端                                 │
│          解析返回的 (u, v, status)                       │
│          更新 shared_state.latest_goal_uv               │
│          更新 shared_state.latest_status                │
│      sleep(0.3s)                                         │
└─────────────────────────────────────────────────────────┘
```

**优势**：
- 主线程不阻塞等待云端响应
- 云端推理延迟不影响机器人控制频率
- 机器人始终基于最新可用的推理结果行动

## 九、评估指标

| 指标 | 定义 |
|------|------|
| **SR** (Success Rate) | 最终停止位置距离目标 < 2.0m 的比例 |
| **OSR** (Oracle Success Rate) | 导航过程中曾经到达过 < 2.0m 的比例 |
| **SPL** (Success weighted by Path Length) | SR × (最短路径 / 实际路径) |
| **NE** (Navigation Error) | 最终位置到目标的距离 |

## 十、总结

V3 版本的核心优势：

1. **出生地环视**：避免一开始走错方向，节省探索时间
2. **FBE 探索**：遇到困境时主动探索新区域，而非原地打转
3. **目标验证**：两阶段验证机制，减少误判
4. **短期记忆**：防止在错误目标附近死循环
5. **位置卡住检测**：即使云端返回有效状态，物理卡住也能脱困
6. **异步通信**：云端延迟不影响机器人响应

这些机制协同工作，使机器人能够在复杂室内环境中稳定、高效地完成视觉语言导航任务。
